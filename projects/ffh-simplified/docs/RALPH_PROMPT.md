C) Content Bible (this is the main missing piece):
   - 8–12 factions: fantasy identity, mechanical hook, strengths/weaknesses, “feel”, unique units/buildings/spells, AI personality.  Note that a given faction will always be EITHER human OR AI.  (We can have a faction be both, but mechanically this will effectively mean creating two different "factions" since mechanically players and AIs will not be doing the same things.)
   - Tech/civics progression: 3–5 eras/tiers, ~40–70 nodes total; each node unlocks specific, named content
   - Magic: 6–10 schools/spheres; each has tiered spells, passive effects, counters
   - Units: core roster + faction uniques; clear roles; upgrade paths; costs; constraints
   - Buildings: city development that supports different playstyles; prerequisites; tradeoffs
   - Resources & terrain: strategic + luxury equivalents; how they matter; map distribution rules
   - Neutral threats: barbarians/monsters, lairs, world bosses (keep “mythic” but manageable)
D) Balance + tuning framework:
   - explicit knobs (numbers you expect to tune)
   - counterplay matrix (faction vs faction, magic vs units, economy vs rush)
E) How The AI Works
   - This is an explicitly asymmetric strategy title, since we will want to give the player far more complex tools to work with than the AI.
   - That means the AI will be using a totally different and much simpler economic system than the player.
   - This economic system needs to be player-legible and player-interactable.
   - We are assuming there will never be a human opponent.
   - The diplomacy also needs to be player-legible and player-interactable.

WHAT ACTUALLY MAKES THE GAME FUN:
The core fantasies/projects the player is embarking on are:
1) Engine-building, where the engine is the civilization and the output is (essentially) a consistently impressive military that is able to both expand and hold onto what it captures.
2) Alliance-forming and grand strategy: we want to capture the feel of "armageddon is coming and the player is attempting a grand crusade to stop this from happening."

PROCESS RULES (Ralph rules):
1) Start by listing the unknowns/gaps in my inputs. Then make reasonable assumptions *explicitly* and keep them consistent.
2) When you invent content, tie it to the loop: every faction/tech/spell must “touch” a system and create decisions.
3) Prefer breadth through combinatorics: fewer base systems, more meaningful content interactions.
4) Avoid bloat: anything that adds complexity must pay for itself in gameplay.
5) Each iteration, run a “design audit” section:
   - missing links (unlocks without purpose, factions without counters, events without triggers)
   - broken incentives (clear degenerate strategies, snowballing)
   - content redundancy
6) Revise the doc to fix audit findings.
